# ScentHive Roadmap

## Phase 1: Make The Core Reliable

- Improve search relevance and aliases.
- Confirm Vercel env vars are present for all environments.
- Add clear errors when search, AI, or scan fails.
- Add a manual fallback for missing fragrance search results.

## Phase 2: Make The Daily Loop Great

- Reduce daily logging friction.
- Make rating, occasion, and memory note feel intentional.
- Improve diary streak and recent-wear feedback.
- Add better empty states for first-time users.

## Phase 3: Clean Up The Codebase

- Split CSS from `index.html` into `styles.css`.
- Split browser JavaScript into `app.js`.
- Extract repeated UI builders into small helper functions.
- Add a lightweight smoke-test checklist.

## Phase 4: Product Polish

- Redesign home into a Spotify/Letterboxd-style fragrance feed.
- Reposition AI as personalization, taste test, and smart fragrance insight instead of a generic chat feature.
- Define a clear ScentHive wordmark and brand system.
- Improve profile and public profile sharing.
- Improve collection/hive browsing.
- Improve wishlist and sample tracking.
- Make AI recommendations more grounded in the user's diary and collection.

## Phase 5: Launch Readiness

- Rotate any keys that were previously exposed.
- Add proper README setup instructions.
- Test auth, diary, collection, search, AI, scan, and public profile on production.
- Add a custom domain when ready.
